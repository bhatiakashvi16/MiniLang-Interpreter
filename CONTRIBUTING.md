# CONTRIBUTING TO MINILANG
Thank you for your interest in contributing to **MiniLang**.  
This project is currently being developed as part of the **Introduction to Open Source Development course** by a team of five members. All contributions follow a structured Git and GitHub workflow to ensure code quality, consistency, and proper version control.  
***
## DEVELOPMENT WORKFLOW
- Create a new feature branch from the latest main branch.  
- Implement the assigned feature or bug fix.  
- Test your changes thoroughly before committing.  
- Commit changes using meaningful commit messages.  
- Push the feature branch to GitHub.  
- Create a Pull Request (PR) for review.  
- After review and approval, merge the Pull Request into the main branch. 
***
## BRANCH NAMING CONVENTION & COMMIT MESSAGE GUIDELINES  
- It's suggested to use descriptive branch names. Ex. - feature/parser, feature/store-command, etc.  
- Follow clear & descriptive commit messages . Ex. - feat: Added Compare Command , feat : Implemented Arithmetic Operations , etc.  
***
## CODING GUIDELINES
- Follow consistent C++ coding standards.  
- Use meaningful variable and function names.  
- Keep functions modular and reusable.  
- Add comments where necessary to explain complex logic.  
- Handle exceptions appropriately.  
- Maintain proper code indentation and formatting.  
***
## TESTING
Before submitting a Pull Request, ensure that:  
- The project compiles successfully.  
- All existing commands execute correctly.  
- Newly added features are tested.  
- No existing functionality is broken.  
- Runtime exceptions are handled properly.  
***
## DOCUMENTATION
Whenever a new feature is introduced, update the relevant documentation, including:
- README.md
- CHANGELOG.md
- Architecture documentation (if applicable)
- Usage examples
***
## PULL REQUEST CHECKLIST 
Before requesting a merge, verify that:  
- The code compiles without errors.
- The feature works as expected.
- Commit messages follow the project convention.
- Documentation has been updated if required.
- No unnecessary files are included in the commit.
